==================================================
ARQUITETURA REAL DE PRODUÇÃO
(RECOMENDADA)
==================================================

Objetivo:
Projetar uma arquitetura enterprise-grade
segregada em múltiplas VPS com foco em:

- segurança
- isolamento
- alta disponibilidade
- escalabilidade
- redução de superfície de ataque
- separação de responsabilidades
- resiliência operacional

Modelo recomendado para evolução futura
do ambiente self-hosted.


==================================================
VISÃO GERAL DA ARQUITETURA
==================================================

Internet
   ↓

Cloudflare
   ↓

VPS 1 — Edge Layer
   ↓ private network ↓

VPS 2 — Identity Tier
   ↓ private network ↓

VPS 3 — PostgreSQL Primary
   ↓ replication ↓

VPS 4 — PostgreSQL Replica

   ↓

VPS 5+ — Aplicações


==================================================
VPS 1 — EDGE LAYER
==================================================

Função:
Camada de entrada da infraestrutura.

Responsável por:
- entrada de tráfego
- proxy reverso
- TLS termination
- proteção edge
- roteamento seguro
- filtragem inicial

Componentes:
- Cloudflare Tunnel
- Nginx
- WAF
- Rate Limiting

Responsabilidades:
- publicar serviços internos
- ocultar infraestrutura crítica
- proteger contra exposição direta
- aplicar políticas HTTP
- bloquear tráfego malicioso
- limitar abuso e brute force

Características importantes:
- única camada exposta externamente
- sem acesso direto ao banco
- comunicação controlada
- hardened reverse proxy
- deny-by-default

Boas práticas:
- fail2ban
- security headers
- TLS moderno
- HSTS
- CSP
- proteção anti-DDoS
- logs centralizados


==================================================
VPS 2 — IDENTITY TIER (CRÍTICA)
==================================================

Classificação:
TIER 0 INFRASTRUCTURE

Esta é a camada mais crítica da empresa.

Comprometimento desta camada:
- compromete autenticação
- compromete identidade
- compromete acesso corporativo
- compromete secrets
- compromete MFA

Componentes:
- Keycloak
- Redis
- SMTP Relay
- Infisical

Responsabilidades:
- autenticação centralizada
- SSO corporativo
- MFA
- gerenciamento de identidade
- gerenciamento de secrets
- integração OAuth2/OIDC/SAML
- integração Microsoft 365

Características importantes:
- isolada da internet pública
- acesso apenas via camada Edge
- comunicação apenas por rede privada
- logs auditáveis
- hardening máximo

Regras de segurança:
- MFA obrigatório
- SSH apenas com chave
- root login desabilitado
- deny-by-default
- sem portas públicas
- segmentação de rede obrigatória
- secrets centralizados
- rotação de credenciais

Importante:
Essa VPS deve ser tratada como:

- ambiente crítico
- infraestrutura sensível
- núcleo de identidade corporativa


==================================================
VPS 3 — POSTGRESQL AUTH PRIMARY
==================================================

Função:
Banco principal da camada IAM.

Componentes:
- PostgreSQL Auth Primary

Responsabilidades:
- persistência do Keycloak
- persistência do Infisical
- armazenamento transacional
- consistência dos dados críticos

Características importantes:
- nunca exposto publicamente
- acesso apenas via rede privada
- backup criptografado
- monitoramento obrigatório
- volumes persistentes

Boas práticas:
- WAL archiving
- backups automáticos
- retenção configurada
- restore validado periodicamente
- tuning PostgreSQL
- auditoria


==================================================
VPS 4 — POSTGRESQL REPLICA
==================================================

Função:
Replica do banco principal para:

- redundância
- recuperação
- failover futuro
- continuidade operacional

Componentes:
- PostgreSQL Replica

Tipo de replicação:
- streaming replication

Responsabilidades:
- replicação contínua
- standby operacional
- disaster recovery
- recuperação rápida

Características importantes:
- sincronização privada
- isolamento de rede
- monitoramento constante
- preparada para promoção futura

Objetivos:
- reduzir downtime
- aumentar resiliência
- preparar failover enterprise-grade


==================================================
VPS 5+ — APPLICATION LAYER
==================================================

Função:
Hospedagem das aplicações corporativas.

Exemplos:
- ERP
- CRM
- Dashboards
- APIs
- aplicações internas

Responsabilidades:
- consumir autenticação do Keycloak
- utilizar secrets do Infisical
- operar desacopladas do IAM

Características importantes:
- isolamento do IAM
- comunicação controlada
- autenticação centralizada
- sem acesso direto aos bancos críticos

Boas práticas:
- OAuth2/OIDC
- JWT validation
- RBAC
- segregação por aplicação
- observabilidade integrada
